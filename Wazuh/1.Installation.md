# 1. Introduction

This document describes the complete deployment of a professional Security Operations Center (SOC) home laboratory using Wazuh on Apple Silicon hardware.

Unlike a simple installation guide, this document explains every engineering decision taken during the deployment process, including architecture, networking, Docker internals, certificate generation, troubleshooting methodology, verification, and operational best practices.

The deployment follows an infrastructure-as-code mindset where every component is reproducible, documented, and version controlled using GitHub.

The objective is to build a reusable SOC environment that can later integrate additional technologies including:

# 4. CyberLab Overview

CyberLab is a personal engineering environment designed to simulate an enterprise Security Operations Center.

Unlike a traditional virtual machine lab, CyberLab uses Docker containers to deploy infrastructure quickly while maintaining reproducibility and portability.

Every deployment is documented.

Every command is version controlled.

Every troubleshooting step is recorded.

Every configuration change is reproducible.

This approach mirrors the operational practices used in modern DevSecOps and SOC engineering teams.

# 5. Why Wazuh?

Wazuh is an open-source Extended Detection and Response (XDR) and Security Information and Event Management (SIEM) platform.

It combines:

• Log Collection

• File Integrity Monitoring

• Vulnerability Detection

• Security Configuration Assessment

• Threat Detection

• Compliance Monitoring

• Active Response

• MITRE ATT&CK Mapping

• Rootkit Detection

• Malware Detection

Wazuh provides centralized visibility across Linux, Windows, macOS, cloud platforms, and containerized workloads.

For a SOC analyst, Wazuh serves as the primary platform for monitoring, detection engineering, and incident investigation.

# 6. Why Docker?

Docker packages an application together with all of its dependencies into a lightweight container.

Instead of installing software directly on the host operating system, Docker isolates each application within its own runtime environment.

Benefits include:

• Faster deployment

• Easy upgrades

• Consistent environments

• Simplified rollback

• Better resource utilization

• Easier troubleshooting

For Wazuh, Docker allows the Manager, Dashboard, and Indexer to operate independently while communicating over virtual networks.

# 7. Why OrbStack?

Apple Silicon uses the ARM64 architecture.

Many traditional virtualization tools were designed primarily for x86 processors.

OrbStack provides a lightweight virtualization layer optimized for Apple Silicon.

Advantages include:

• Native ARM support

• Faster startup

• Lower memory consumption

• Better filesystem performance

• Integrated Docker runtime

• Excellent macOS integration

OrbStack enables enterprise container workloads without requiring Docker Desktop.

# 8. Apple Silicon Architecture

Apple Silicon differs significantly from Intel processors.

Architecture:

Apple M5

↓

ARM64

↓

macOS

↓

OrbStack Virtualization Layer

↓

Docker Engine

↓

Docker Containers

↓

Wazuh Services

ARM-native containers generally consume less power while delivering excellent performance.

When selecting Docker images, ARM64 compatibility should always be verified.
